**Use-Case Diagram for GymFlow System**

I have created a use-case diagram for the GymFlow gym management system. The diagram includes three main actors (Member, Trainer, Administrator) and 26 use cases covering all system functionality.

**Actors:**
- **Member**: Gym members who view workout plans and register for classes
- **Trainer**: Fitness trainers who create workout plans and manage classes  
- **Administrator**: System administrators who manage users and equipment

**Key Use Cases:**
- Member: Login, View Workout Plan, View Class Schedule, Register for Class, View Attendance History
- Trainer: Login, Create/Assign Workout Plans, Manage Class Sessions, Record Attendance, Import/Export Templates
- Administrator: Login, Manage Users, Manage Equipment, View Reports, Export Data
